# Teoria
Mikä on Git?

Perustuu kirjaan Pro Git

Git on versionhallintajärjestelmä, jolla seurataan tiedostojen muutoksia
Git tallentaa tiedot snapshotteina (tilannekuvina)
Git on hajautettu, eli jokaisella käyttäjällä on koko projektin historia
Git toimii paikallisesti ilman internetiä
Git käyttää tarkistussummia (hash), mikä tekee siitä luotettavan
Oma havainto

Git toimii kuin aikakone, josta voi palata aiempiin versioihin.

## Komennon selitys
git add --all && git commit; git pull && git push


git add --all
lisää kaikki muutokset staging-alueelle

git commit
tallentaa muutokset Gitin historiaan

git pull
hakee muutokset palvelimelta ja yhdistää ne

git push
lähettää muutokset GitHubiin
&&
seuraava komento suoritetaan vain jos edellinen onnistuu
;
seuraava komento suoritetaan aina

## d) Tukki

Tarkastelin repositoryn lokia:

git log

git log --oneline

git log näyttää commit-historian yksityiskohtaisesti

git log --oneline näyttää lyhyen version

Lokista näkee commitit, tekijän, sähköpostin ja ajan. Huomasin, että commit-historiassa näkyi eri tekijätietoja, koska asetuksia oli muutettu.

## e) Gitanbile

Loin ansible-kansion ja lisäsin playbook.yml-tiedoston:

mkdir ansible
nano ansible/playbook.yml

Lisäsin sen Git-versionhallintaan:

git add .

git commit -m "Add ansible playbook"

git push

Tämän jälkeen muokkasin tiedostoa ja tein uuden commitin:

git add .

git commit -m "Update ansible playbook"

git push

Näin muutokset tallentuivat Gitin historiaan.
